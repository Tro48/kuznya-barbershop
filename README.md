# Кузня — лендинг барбершопа

**Демонстрационный проект.** Компания «Кузня», мастера, цены и отзывы вымышлены.
Лендинг сделан как портфолио-работа, а не по заказу реального бизнеса.

> Статус: каркас репозитория. Разработка не начата.

## О проекте

Лендинг барбершопа с одной целью: заявка на запись с минимальным трением.
Заявка уходит в Telegram администратору, а не «в никуда» — это то, что отличает
работающий лендинг от вёрстки.

Визуальная идея — **мастерская, а не «винтажный джентльмен»**: приглушённая латунь
вместо золота, узкий гротеск капслоком, чертёжные линии, тёмная тема без светлого
варианта. Никаких иконок с ножницами и усами.

## Что здесь показано технически

- **Route Handler с серверной валидацией** той же Zod-схемой, что и клиентская форма.
  Клиентская проверка — это UX, а не защита.
- **Токен бота недоступен из браузера** — `import "server-only"`, проверяемо поиском
  по собранному бандлу.
- **Отказоустойчивость**: если Telegram API недоступен, заявка пишется в Supabase,
  и пользователь всё равно видит успех. Заявка не теряется.
- **Идемпотентность**: повторный клик по кнопке не создаёт вторую заявку.
- **Защита формы** без капчи: honeypot, time-trap, rate limit по IP.
- Карта Яндекса грузится по клику, а не сразу — иначе она весит больше, чем весь
  остальной лендинг.

## Стек

Next.js (App Router) · TypeScript · Tailwind CSS · Motion · React Hook Form · Zod ·
Supabase · Telegram Bot API · Vercel

## Как начать

Репозиторий пока содержит только документацию и конфигурацию. Скаффолд:

```bash
pnpm create next-app@latest . --typescript --tailwind --app --eslint --src-dir
cp .env.example .env.local   # и заполнить
pnpm dev
```

Версии зависимостей фиксируются в lock-файле на старте работы и не обновляются по
мажорам до сдачи проекта.
